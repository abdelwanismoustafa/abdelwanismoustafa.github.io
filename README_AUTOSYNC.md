# ORCID auto-sync for Academic Pages

This pack makes your publication list and homepage publication counters update automatically from your public ORCID record.

## Files to copy into your repo

- `.github/workflows/sync-publications-and-deploy.yml`
- `scripts/sync_publications.py`
- `_data/site_profile.yml`
- `_data/publication_overrides.yml`
- `_pages/about.md`
- `_pages/publications.md`

## One-time setup in GitHub

1. Go to **Settings → Pages**.
2. Under **Build and deployment → Source**, select **GitHub Actions**.
3. Go to **Settings → Secrets and variables → Actions**.
4. Add these repository secrets:
   - `ORCID_CLIENT_ID`
   - `ORCID_CLIENT_SECRET`

## How to get the ORCID secrets

1. Sign in to your ORCID account.
2. Open **Developer Tools**.
3. Register a **Public API** client.
4. Copy the client ID and client secret into your GitHub repository secrets.

## How the sync works

- The workflow runs on every push to `main`.
- It also runs once per day on a schedule.
- The Python script reads your public ORCID works.
- If a work has a DOI, the script enriches it with Crossref metadata.
- The script rewrites `_data/publications.yml`.
- If that file changed, the workflow commits it back to the repo.
- The same workflow then builds and deploys the site to GitHub Pages.

## Where to edit static information

- Update awards, hero text, and profile links in `_data/site_profile.yml`.
- Update hand-written descriptions, themes, highlights, and extra links in `_data/publication_overrides.yml`.

## Important note

Do not hand-edit `_data/publications.yml`. That file is generated by `scripts/sync_publications.py`.

Use `_data/publication_overrides.yml` for manual refinements instead.
